## Qu’est‑ce que Sushi SaaS ?

Sushi SaaS est un starter Next.js pragmatique, orienté production, pour construire et lancer de vrais produits SaaS. Il fournit dès le départ l’authentification, la facturation, l’admin, le contenu et l’i18n, reliés à une base typée avec des migrations explicites.

Pile principale :

- Next.js App Router avec routes localisées (next‑intl)
- Better Auth + Postgres via Drizzle ORM
- Stripe Checkout + finalisation par webhooks (journal de crédits prêt)
- Docs/blog MDX avec Fumadocs (métadonnées SEO + JSON‑LD)
- Pages d’admin avec RBAC côté serveur
- DX saine : pnpm, ESLint/Tailwind, endpoint de santé, variables d’environnement claires

Dépôt GitHub : https://github.com/PansaLegrand/saas-sushi-template

```bash
git clone https://github.com/PansaLegrand/saas-sushi-template.git
cd saas-sushi-template
pnpm install && pnpm dev
```

---

## Pourquoi vous pouvez lui faire confiance

- Typage de bout en bout : les schémas Drizzle sont la source de vérité ; les migrations drizzle‑kit gardent la base en phase.
- Vrais parcours, pas des mocks : auth, facturation (Stripe) et permissions admin sont connectées et visibles dans l’UI.
- i18n prévisible : routes préfixées par locale via next‑intl ; messages au même endroit que l’app et les docs.
- Clarté opérationnelle : `/api/health`, modèles d’environnement et consignes de build/lint/migrations.
- Documentation claire : chaque capacité (auth, billing, base, admin, e‑mails) a un guide MDX avec commandes et chemins utiles.
- Éprouvé sur le terrain : reflète des patterns utilisés pour livrer de vrais produits (voir README « Showcase »).
